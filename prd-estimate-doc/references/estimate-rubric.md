# Estimate Rubric

## 1. 真人开发口径

适用条件：
- 前端自己完成需求理解、开发、联调、自测、回归
- AI 只做辅助，不直接承担主要实现

建议包含：
- 页面开发
- 复杂交互
- 接口联调
- 自测回归
- 风险冗余

## 2. AI 协同开发口径

适用条件：
- 用户给 PRD / 边界
- AI 负责大部分代码实现和文档整理
- 用户负责关键边界确认、review、真实联调

建议调整：
- 压缩纯开发工时
- 保留联调、自测、回归的必要成本
- 明确哪些任务由 AI 直接实现

## 3. 叶子任务估时规则

推荐：
- 文档口径：可按模块汇总
- Base 口径：叶子任务尽量 `<= 4h`

经验规则：
- 纯展示、只读区块：`1~2h`
- 普通列表/筛选/分页：`2~3h`
- 中等复杂表单区块：`2~4h`
- 复杂交互、SKU、差量回显、提交链路：拆成多个 `2~4h` 子任务
- 接口联调：单独成项
- 全链路回归：单独成项

## 4. 并行和波次

建议字段：
- `执行方式`：`可并行` / `必须串行`
- `执行波次`：整数 `1/2/3/4/5`

推荐语义：
- Wave 1：基础接入、列表参数、公共能力
- Wave 2：可并行的主体开发
- Wave 3：依赖前序产物的串行开发
- Wave 4：联调/自测
- Wave 5：全链路回归

## 5. 常见低估点

- 长表单只按一个页面估时
- 忽略规格、SKU、价格规则、模板互斥等复杂交互
- 把联调和回归吸进开发工时
- 父任务、子任务同时计时
- 需求边界未确认却给出精确结论
