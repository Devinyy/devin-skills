# PRD Estimate Doc 对外发布说明

各位同事，

这里提供一个可直接复用的 `prd-estimate-doc` skill，用于把一份 PRD 快速转换成研发侧可执行产物，减少重复读 PRD、重复拆任务和反复对估时口径的成本。

## 这个 skill 解决什么问题

它适合在拿到 PRD 后，快速产出以下内容：

- PRD 结构化摘要
- 前端任务拆解
- 前端技术方案
- 研发估时文档

如果你们正在用飞书 Base 管理排期，它还支持继续把结果同步成：

- 父任务 / 子任务结构
- 开发 / 联调拆分
- `AI直接实现`
- `执行方式`
- `执行波次`

## 适用场景

- 需求评审前，需要快速形成前端评估
- 需要区分真人开发口径和 AI 协同开发口径
- 需要把任务拆到叶子粒度并直接落到 Base
- 需要把估时过程沉淀成可重复复用的标准动作

## 包内包含什么

- `SKILL.md`
  定义 skill 的适用时机、工作流和产物要求
- `README.md`
  安装说明、使用方式和脚本前置条件
- `references/`
  工作流、估时口径、Base 同步说明、发布检查清单
- `templates/`
  Base 脚本配置模板
- `scripts/`
  Base 批量重写和 AI 排期字段脚本

## 安装方式

把目录安装到：

```text
~/.agents/skills/prd-estimate-doc
```

如果拿到的是压缩包，可以直接解压到上面的目录。

## 推荐触发方式

- “用 `prd-estimate-doc` 根据这个 PRD 出估时文档”
- “按 `prd-estimate-doc` 把这份 PRD 同步成 Base 估时表”
- “按 `prd-estimate-doc` 给我一版 AI 协同开发口径估时”

## 使用前提

- 本机已安装 `lark-cli`
- 已完成飞书授权
- 如果涉及 Base 回写，已具备目标 Base 的访问权限

## 注意事项

- 压缩包里不应包含真实 `baseToken`、真实表 ID、真实用户数据
- 运行脚本前请先看 `templates/*.example.json` 和 `references/base-sync.md`
- 真正回写 Base 前，建议先跑一次只读预览

## 当前版本

- 版本：`1.1.0`
- 能力范围：PRD 摘要、任务拆解、技术方案、估时文档、Base 批量同步

如果后续需要扩成“后端方案 + 联调排期 + 多角色协作”的版本，可以在这个 skill 基础上继续扩展。
