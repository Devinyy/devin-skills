# Style Guide

## 目标风格

参考图是“横向分层能力地图”，不是流程图。核心视觉语言：

- 横向宽画布
- 顶部图例
- 黑色标题条
- 左侧层级栏
- 每一层横向铺开能力域
- 大能力域用浅蓝容器
- 具体功能用小矩形卡片
- 底部用圆角卡片承载中台能力、开发域、运维域、运营域

## 颜色语义

| 状态 | 推荐色 | 用途 |
| --- | --- | --- |
| 已实现 | `#5AFF88` | 已完成能力 |
| 建设中 | `#5AFFFF` | 进行中能力 |
| 未建设 | `#BBBFC4` | 未开始能力 |
| 即将/已废弃 | `#000000` | 废弃或即将下线能力 |
| 其他部门支持 | `#FFA800` | 外部团队提供能力 |
| 标题/域标签 | `#000000` | 主标题、域标题 |
| 业务域容器 | `#D6DCF3` | 业务分组背景 |
| 层级栏 | `#F0F4FC` | 左侧层级标签 |
| 已实现业务卡 | `#5178C6` | 业务层已实现模块 |

## 布局规则

### 顶部

- 左上或顶部放“图例”
- 图例依次展示：已实现、建设中、未建设、即将/已废弃、其他部门支持
- 主标题用黑底白字长条，例如：“我店生活大前端业务架构”

### 主体层级

建议从上到下：

1. 宿主层
   - 放 App、小程序、商家端、平台端、运营端等入口
2. 业务层
   - 放电商、本地生活、直播、营销、商品、用户、交易、配置中心等业务域
3. 组件层
   - 放开放平台、基础组件、支付、商品卡片、SKU、地图、表单、弹窗等
4. 基础层
   - 放统一请求库、全局路由、APM、应用通信、版本管理等
5. 大前端中台能力
   - 作为下半区分隔标题
6. 开发域
   - 脚手架、微前端、工具套件、UI 组件库、业务组件库、网络请求库
7. 运维域
   - 自动化编译打包、灰度发布、热修复、性能监控、性能分析
8. 运营域
   - IM、活动营销、版本更新、推送等

## 文案规则

- 卡片文案尽量短，2 到 6 个字最佳
- 能力域用名词，不写完整句子
- 不在卡片里写解释段落
- 复杂说明放在图后面的“说明”章节

## 图形规则

- 层级标签使用浅色矩形
- 业务域容器使用浅蓝背景
- 业务功能卡使用深蓝或状态色
- 中台能力卡使用圆角矩形
- 尽量避免大量连线
- 如果必须表达依赖，只画少量关键箭头

## 信息组织规则

先分层，再分域，再放能力：

```text
层级
  能力域
    能力项
```

不要直接把所有能力摊平成同一层。

## 最小验收标准

- 有图例
- 有主标题
- 有至少 3 个层级
- 每个能力项有状态颜色
- 业务能力和基础能力分开
- 不是普通 flowchart 样式
